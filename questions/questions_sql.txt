1. Что такое SQL? Для чего он применяется? 

    Ответ для собеседования
    SQL (Structured Query Language) — это язык запросов, предназначенный для работы с реляционными базами данных. 
    Он используется для извлечения, добавления, обновления и удаления данных, а также для управления структурами базы данных.

    Подробный ответ
    SQL (Structured Query Language) — это стандартный язык, используемый для взаимодействия с реляционными базами данных. 
    Он позволяет выполнять широкий спектр операций, включая создание и модификацию структуры базы данных (DDL - Data Definition Language), 
    управление данными (DML - Data Manipulation Language), и контроль доступа к данным (DCL - Data Control Language).

    Основные команды SQL включают:

    SELECT: используется для извлечения данных из базы данных.
    INSERT: используется для добавления новых записей в таблицу.
    UPDATE: используется для изменения существующих записей.
    DELETE: используется для удаления записей из таблицы.
    CREATE TABLE: используется для создания новой таблицы.
    ALTER TABLE: используется для изменения структуры существующей таблицы.
    DROP TABLE: используется для удаления таблицы.


2. В чем отличие между DDL, DML, DCL и TCL? Какие операции входят в каждую из групп?

    Data Definition Language (DDL, язык описания данных) — позволяет выполнять различные операции с базой данных, такие как CREATE (создание), ALTER (изменение) и DROP (удаление объектов).
    Data Manipulation Language (DML, язык управления данными) — позволяет получать доступ к данным и манипулировать ими, например, вставлять, обновлять, удалять и извлекать данные из базы данных. 
    Самые используемые: INSERT, SELECT, UPDATE, DELETE.
    Data Control Language (DCL, язык контролирования данных) — позволяет контролировать доступ к базе данных. Пример — GRANT (предоставить права), REVOKE (отозвать права).
    Transaction Control Language (TCL, язык контролирования транзакций) - позволяет управлять транзакциями. COMMIT, ROLLBACK, SAVEPOINT - примеры операторов TCL.


3. Что такое СУБД?

    Система управления базами данных (СУБД) — программное обеспечение, которое взаимодействует с пользователем, приложениями и самой базой данных для сбора и анализа данных. 
    СУБД позволяет пользователю взаимодействовать с базой данных. Данные, хранящиеся в базе данных, могут быть изменены, извлечены и удалены. 
    Они могут быть любых типов, таких как строки, числа, изображения и т. д.

    Не путать с базой данных! База данных — структурированная коллекция данных.

    Существует два типа СУБД:

    Реляционная: данные хранятся в отношениях (таблицах). Примеры — PostgreSQL, MySQL.
    Нереляционная: не существует понятия отношений, кортежей и атрибутов. Примеры— MongoDB, Cassandra.


4. В чем разница между типом данных CHAR и VARCHAR в SQL?

    CHAR и VARCHAR служат символьными типами данных:
    VARCHAR используется для строк символов переменной длины
    CHAR используется для строк фиксированной длины

    Например, char(10) может хранить только 10 символов и не сможет хранить строку любой другой длины (если попытаться добавить строку меньшей длины оставшееся место будет 
    заполнено пробелами), тогда как varchar(10) может хранить строку любой длины до 10, т.е. например 6, 8 или 2.


5. Что такое индексы в SQL? Зачем они нужны?

    Ответ для собеседования
    Индексы в SQL - это структуры данных, используемые для ускорения выполнения запросов к базе данных. Они создаются на столбцах таблицы для быстрого доступа к данным.

    Подробный ответ
    Индексы в SQL играют ключевую роль в оптимизации производительности запросов к базе данных. Они представляют собой структуры данных, которые сортируют и организуют 
    информацию в таблицах. 
    Создание индексов на часто используемых столбцах позволяет существенно ускорить выполнение запросов, так как база данных может быстро находить нужные строки без 
    необходимости сканировать всю таблицу.

    Важно понимать, что индексы не только ускоряют операции чтения данных, но также могут повысить производительность операций обновления и удаления. 
    Например, если в запросе используется условие WHERE, индекс на соответствующем столбце позволит базе данных быстрее находить строки, которые нужно изменить или удалить.

    Однако создание слишком большого количества индексов также может иметь негативное влияние на производительность, так как это увеличивает время на обновление индексов при 
    изменении данных в таблице. Поэтому важно находить баланс между количеством индексов и производительностью базы данных, оптимизируя их использование в зависимости от 
    конкретных запросов и требований к приложению.


6. Что такое представление (VIEW) в SQL и как оно используется?

    Представление (VIEW) — это виртуальная таблица, которая является производной от одной или нескольких таблиц в базе данных. 
    Такие виртуальные таблицы используются для упрощения сложных запросов, представляя данные в более организованном и осмысленном виде. 
    Они также могут использоваться для соблюдения политики безопасности, скрытия конфиденциальной информации и сокращения объема данных, которые необходимо извлекать из БД.

    Для создания представления можно использовать оператор CREATE VIEW.

    Представления SQL обеспечивают ряд преимуществ для администраторов баз данных и разработчиков:

    Упрощение сложных запросов. Представления позволяют скрывать сложность объединений, подзапросов и других продвинутых функций. Это облегчает написание и сопровождение SQL-запросов с течением времени.
    Усиление безопасности. Представления можно использовать для обеспечения соблюдения политики безопасности, скрывая конфиденциальную информацию от пользователей, не имеющих разрешения на доступ к ней.
    Улучшение производительности. С помощью представлений можно уменьшать объем данных, которые необходимо извлечь из БД, отфильтровывая всё лишнее. Это позволяет повысить производительность запросов и снизить нагрузку на сервер.
    Уменьшение дублирования кода. Представления можно использовать для устранения дублирования кода, предоставляя единый источник информации для сложных SQL-запросов. Это облегчает сопровождение и обновление кода SQL с течением времени.
    Упрощение анализа данных. Представления помогают более наглядно представлять данные аналитикам и бизнес-пользователям. Это позволяет лучше понять сложные взаимосвязи между данными и принять более обоснованные решения.


7. Объясните разницу между первичным и внешним ключом.

    Первичный ключ (primary key) — это столбец или комбинация столбцов, которые уникально идентифицируют каждую строку в таблице. 
    Он обеспечивает целостность и уникальность данных в пределах таблицы. У каждой таблицы может быть только один первичный ключ.

    Внешний ключ (foreign key) — это столбец или комбинация столбцов, которые ссылаются на первичный ключ другой таблицы. 
    Он устанавливает связь между двумя таблицами и обеспечивает ссылочную целостность. 
    Таблица может иметь несколько внешних ключей, указывающих на её связь с разными таблицами.


8. Что такое уникальный ключ (Unique key)?

    Уникальный ключ — это группа из одного или нескольких полей или столбцов в таблице, которые однозначно идентифицируют запись базы данных. 
    Уникальный ключ нужен для обеспечения неповторяемости значений в таблице.


9. В чем разница между операторами DELETE и TRUNCATE?

    DELETE используется для удаления 1-N строк в таблице, а TRUNCATE для удаление всех данных в таблице
    После DELETE можно восстановить данные, а после TRUNCATE нельзя (исключение SQL Server)
    DELETE - DML команда, а TRUNCATE - DDL
    Оператор TRUNCATE работает быстрее
